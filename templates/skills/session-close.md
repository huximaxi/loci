# Session Close

*End on purpose, not on exhaustion. A session's learning has a half-life, and
it is freshest at the close; deferring consolidation to "later" is where the
learning leaks.*

---

## What it does

Turns the end of a working session into a small, fixed discipline: reflect on
what shifted, adjust the working docs while the shifts are still visible,
bank the durable learning, and write a short honest record. The close is not
an optional extra after the real work; it is the step that makes the next
session start warm instead of cold.

The failure it prevents is quiet: a session that ends without this pass loses
exactly the things that were hardest to earn (the decision's *why*, the
pattern that finally got named, the loose end that everyone still remembered
at the time).

## When to apply

Any time a working session ends with intent to stop: an explicit "let's
wrap", a natural completion, or a handover to a future session. This is
closure's standard form. A hard interrupt (crash, timeout, emergency) is the
only exempt ending, and the next session's open should then run this pass
retroactively, first thing.

## The procedure

1. **Reflect.** What shifted this session? New learnings, decisions made and
   their reasons, patterns observed, things that grew or died. List them
   before touching any file; the list is the input to everything below.
2. **Adjust the docs.** Update the current-focus note, prune done items from
   the quest log or task list, fix anything in persona or skill files that
   drifted from how the work actually went. Leave untouched what doesn't
   need it; a close that rewrites everything is as suspect as one that
   rewrites nothing.
3. **Persist.** Run `insight-consolidation.md` on anything durable from step
   1. One fact per note, linked, indexed.
4. **Write the close note.** Three short sentences: what you came with, what
   happened or pivoted, what crystallised or carries forward. Grounded and
   dry, not a victory lap. The open intent was a guess; this is the record.
5. **Confirm.** State plainly what was updated and what was left alone, and
   why. Queue anything that belongs to the next session rather than forcing
   it now.

## Verification

- The next session can open cold, read the close note plus the adjusted
  docs, and know where to stand within a minute or two.
- Every durable learning from step 1 exists as a note (step 3) or was
  deliberately judged ephemeral. None are only in the transcript.
- The quest log contains no item that was actually finished this session.

## Kill condition

This one stays; it is load-bearing for the whole memory loop. If the closes
start feeling like ceremony (nothing shifted, nothing to adjust, notes that
never get recalled), the problem is upstream: sessions too small to learn
anything, or an index too noisy to recall from. Fix those; don't drop the
close.
