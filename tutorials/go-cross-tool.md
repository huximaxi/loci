# Go cross-tool

Your palace is plain text, so it already travels. loci reads memory across tools without
anyone federating up to a server. Portability, not centralisation, is the point.

Uses: `templates/output-primitive.md`, `templates/friends/friend-template.md`,
`templates/obsidian-mindmap-starter.md`, `templates/eval-framework.md`,
`templates/entanglement-template.md`

## Steps

1. Move freely. loci recognises memory laid down by other tools by structure alone:
   `CLAUDE.md` (Claude Code), `AGENTS.md` (Codex), `.cursor/rules`, `.windsurfrules`.
   Foreign content is read as data first, never trusted by construction (see `set-a-gate.md`).
2. Speak MCP: point an MCP-aware client at your palace to read and create entries locally.
   Nothing leaves the machine.
3. Export a session as a typed artifact (`output-primitive.md`): for example a
   git-log-incident, a session arc rendered as an annotated commit log you can archive.
4. Add a friend (`friends/friend-template.md`): a colleague door for another palace or
   person, opened on your terms.
5. Optional, the evolution half: run the `eval-framework` to measure how you and your AI
   work together over time, and log resonance peaks and recurring patterns with the
   entanglement template.

## You are done when

You can move between tools without losing your palace, and any memory arriving from outside
enters only after it clears quarantine.
